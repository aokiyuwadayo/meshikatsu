# ① 方向性すり合わせ：A/B比較メモ（判断材料）

> 担当レーン: 「方向性すり合わせ＋フロント」/ 提出先: リーダー・PO・全員
> 目的: いま並走している **2つのバージョン**のどちらで行くか（or 折衷）を、5分で決められるようにする。
> ⚠️ このメモは**判断材料**です。最終決定はリーダー/POが下してください。

---

## 0. 結論サマリ（先に読む）

- **A（HTML版）** と **B（Next.js版）** は、見た目が似ているが **ビジネスの根っこが別物**。
  - A = **食品を売る**（自社の冷凍食品を ¥390/食で販売＋食べるタイミング管理）→ 物販/D2C
  - B = **ユーザーの食品ロスを減らす道具**（私物の食材を管理・AI・ゲーム化）→ ツール/SaaS
- だから論点は「Aの機能を足すかBの機能を足すか」ではなく、**「我々は“食品を売る会社”なのか“ロスを減らす道具を作る会社”なのか」**。
- **エンジニアからの推奨は C（折衷）**。Bのツール体験で課題を可視化し、足りない食材をAの¥390商品でレコメンド販売する。課題→解決→収益が一気通貫で、ビジコンで一番強い。ただし採否はリーダー判断。

## 2. それぞれの強み・弱み（ビジコン目線）

### A：HTML版
- **強み**: もう動いている／デプロイ済み／課金モデルが具体的／実装がシンプルで壊れにくい。
- **弱み**: AI・ゲーム性なしで**差別化が弱い**（競合の宅食と土俵が近い）。在庫＝自社商品なので「自分の食材ロスを減らす」体験ではない＝事業計画書 §2 のペイン（仕送り野菜を腐らせる）に直接刺さらない。

### B：Next.js版
- **強み**: AI×ゲーム性×無料の**3点セットが事業計画書 §7 の差別化そのもの**。ユーザーの私物食材に踏み込むのでペインに直撃。拡張性あり。
- **弱み**: 未デプロイ／課金が後付け（今すぐの売上ストーリーが弱い）／キャラアート未完（→③で対応）／実機・実レシート検証が未（→②で対応）。

## 3. 選択肢（リーダーはこの3つから選ぶ）

- **選択肢A：HTML版で行く** … 「今週末に確実に動くものを見せる」を最優先。AI/ゲームは捨てる。
- **選択肢B：Next.js版で行く** … 差別化と拡張性を取る。デプロイ・キャラ・検証を今週末で仕上げる必要あり。
- **選択肢C：折衷（推奨）** … Bをメイン体験にし、Aの¥390商品を「期限切れそう／在庫が薄い時のおすすめ補充」として差し込む。
  - 体験: 冷蔵庫を見る → 「ナスがあと2日」→ AIが旬レシピ提案 → 足りない食材は¥390商品でワンタップ補充。
  - 利点: 課題（ロス）→ 解決（管理・AI）→ 収益（補充販売）が1画面で繋がる。ビジコン審査で一番説明しやすい。
  - コスト: B採用＋Aの商品データを1リストに繋ぐだけ（折衷の技術設計は別途、希望あれば先回りで出します）。
